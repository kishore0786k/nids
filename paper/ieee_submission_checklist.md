# IEEE Submission Checklist

## Software Package

- Backend logic is isolated in `backend/nids_engine.py`.
- Flask API routes are isolated in `backend/app.py`.
- Frontend dashboard assets are isolated in `frontend/`.
- Regression tests exist in `backend/test_smoke.py`.
- Publication artifacts are generated by `backend/generate_publication_package.py`.

## Research Integrity

- Confirm saved-paper-summary metrics are not mixed with live backend metrics.
- Confirm train/test split and preprocessing are documented.
- Confirm all reported numbers match generated JSON/figures.
- Confirm symbolic rule definitions are stated in the method section.

## Reproducibility

- Archive `results/publication_package/publication_package.json`.
- Archive generated PDF figures from `paper/figures/generated/`.
- Include model, dataset preprocessing, and random seed details.
- Re-run `venv\Scripts\python.exe -m unittest backend.test_smoke` before final submission.

## Manual IEEE Work Still Required

- Add final author names, affiliations, funding, and acknowledgments.
- Add verified IEEE references/BibTeX.
- Add full manuscript narrative and limitations.
- Confirm figures are legible in grayscale and in two-column format.
